# MVP 验收指标与测试样例

## 1) 验收指标

- 响应时延：单轮问答 P95 <= 10 秒（本地网络正常场景）。
- 结构化输出达标率：`【要点速览】+【正文解读】+【阅读建议】` 完整率 >= 90%。
- 快捷动作可用性：4 个快捷动作连续触发成功率 >= 95%。
- 作用域正确性：已选论文会话中，回答引用跨论文内容比例 <= 5%。
- 上下文利用率：提供“当前页/选中文本”后，回答显式引用上下文比例 >= 80%。

## 2) 测试样例集（首批）

每条样例建议记录：论文ID、问题、是否使用快捷动作、是否附带页码/片段、耗时、结果评级。

### Case A：3分钟速读

- 输入：点击 `3分钟速读`。
- 期望：
  - 输出三段结构完整。
  - 内容覆盖“研究问题/方法/贡献”。

### Case B：章节摘要

- 输入：点击 `章节摘要`。
- 期望：
  - 按章节或论文结构给出分段摘要。
  - 每段摘要长度适中，可直接复制到笔记。

### Case C：术语解释（带选中文本）

- 输入：阅读区填写选中文本，再点击 `术语解释`。
- 期望：
  - 解释至少 2 个术语。
  - 明确说明术语在该片段中的语境。

### Case D：关键结论提取（带当前页）

- 输入：阅读区设置当前页，点击 `关键结论提取`。
- 期望：
  - 提供可操作结论列表。
  - 回答体现当前页上下文（或明确说明无法定位）。

### Case E：自由提问（paper scoped）

- 输入：`这个方法和传统基线相比优势是什么？`
- 期望：
  - 回答主要来自当前选中文献。
  - 未知内容要显式说明“不确定/未检索到”。

## 3) 回归检查（每次发布前）

- 类型检查：`npm run typecheck`
- 代码规范：`npm run lint`
- 手工冒烟：
  - 导入新PDF后状态从 `索引中` 到 `可问答`。
  - 切换不同论文后问答历史隔离正确。
  - 阅读上下文同步后，AI侧能看到上下文字段。
